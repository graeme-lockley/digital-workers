# Architecture Decision Records

ADRs capture decisions whose rationale future contributors must understand. Each ADR is short, dated, and immutable once accepted (superseded by a later ADR if reversed).

## Index

_None yet._

## Format

```
# NNNN — Title

- **Status:** Proposed | Accepted | Superseded by NNNN
- **Date:** YYYY-MM-DD
- **Context:** Why this decision is needed.
- **Decision:** What was decided.
- **Consequences:** Trade-offs accepted.
```

## Conventions

- Filenames: `NNNN-kebab-title.md` (zero-padded sequential).
- Add an entry to the index above when accepted.
- Reference relevant specs and architecture sections.
